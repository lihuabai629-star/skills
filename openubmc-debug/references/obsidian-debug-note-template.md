# openUBMC Debug 排障记录模板

> [!summary]
> 目标不是记录流水账，而是把可复盘的排查顺序、方法和证据写清楚，让后续可以沿同一路径继续定位。

## 什么时候记一版阶段性总结

- 是否开始写这份总结，由用户决定；用户没有明确要求时，不主动补笔记
- 用户明确要求时，默认在任务完成后写最终总结
- 如果用户明确要求时主路径还在推进，不要边排障边补完整笔记；只在以下情况写一版阶段性总结：
  - 已给出明确结论，需要先交付给用户
  - 形成稳定方法，后续只是在补充证据或验证细节
  - 被外部条件卡住，需要暂停或交接
- 不必等到 100% 根因闭环；只要这一阶段已经能交付思路、方法和下一步，就可以写
